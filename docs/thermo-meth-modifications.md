# Thermo `meth-modifications` Notes

Repository: <https://github.com/thermofisherlsms/meth-modifications>

## What It Provides

Thermo's repository provides XML-based method modifications for TNG 1.2 and higher. It includes:

- `XmlMethodChanger`, a C# wrapper/CLI around Thermo TNG MethodXML libraries.
- Small Thermo MethodXML DLLs under `lib/`.
- Template `.meth` files and XML modification examples for Fusion, Exploris, and TSQ families.
- XSDs describing supported method-modification XML.
- MIT license.

Useful CLI operations from the source:

```text
status
validate <METHOD|XML>
-i template.meth -m mods.xml -o result.meth
-e method.meth -o method.xml
--summary -i method.meth
-c method.xml --model="TSQEndura"
```

## What It Seems Good For

This is useful for mass spectrometer method work:

- modify a template `.meth` with XML
- validate Thermo `.meth` and modification XML
- export Fusion/Tribrid methods to XML
- print a method summary
- modify scan-node parameters
- modify target/inclusion/exclusion mass lists
- modify DIA-related scan settings
- set experiment start/end times

This could become an Aurelius export backend for method pieces such as:

- DIA window lists
- targeted inclusion lists
- GoDig-style target lists
- MS2/MS3 scan settings or target scheduling

## What It Does Not Solve By Itself

The inspected schemas and examples are MS-method oriented. They include terms such as `FullMSScan`, `TMSnScan`, `DIAScan`, `MassList`, `MassListRecord`, `StartTime`, `EndTime`, `StartTimeMin`, `EndTimeMin`, and `LCPeakWidth`, but not LC pump gradient waypoint tables.

So this repository should not be treated as the LC-gradient extractor for arbitrary binary `.meth` files. It may help export a Fusion/Tribrid method to XML, but we still need to verify whether the exported XML contains LC gradient information for methods generated by the user's LC stack.

## Integration Plan

1. Keep LC gradient import separate from MS method modification.
2. For `.meth` source-gradient import, first try supported Thermo export paths:
   - `XmlMethodChanger -e method.meth -o method.xml --model ... --version ...`
   - `XmlMethodChanger --summary -i method.meth --model ... --version ...`
3. Parse exported XML or summary text only if LC gradient waypoints are actually present.
4. Use this repo later as an MS method export backend for DIA and targeted/GoDig modes.
5. For MS1-style optimization, keep using RAW/mzML/feature tables as the observed feature source.

## Local Requirements To Test

The C# wrapper depends on Thermo TNG MethodXML infrastructure and installed instrument model/version registration. A local test should check:

- whether `status` finds the relevant instrument model
- whether the user's `.meth` validates
- whether `--summary` runs on the example `.meth`
- whether `-e` can export the example `.meth` to XML
- whether the resulting XML includes LC gradient waypoints
